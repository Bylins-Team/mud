# ISSUE-016: Неправильный порядок аргументов в sprintf - потенциальная format string уязвимость

**Тип**: Format String / Logic Error
**Критичность**: MEDIUM
**Файл**: `src/administration/punishments.cpp:140`
**Статус**: Активна

## Описание

В функции `SetHell()` на строке 140 обнаружена ошибка в порядке аргументов sprintf, которая может привести к неожиданному поведению или format string уязвимости:

```cpp
sprintf(buf, "%s%s отправил$G вас в темницу.%s", GET_NAME(ch), kColorBoldRed, kColorNrm);
```

Проблемы:
1. **Неправильный порядок**: `GET_NAME(ch)` находится до цветового кода, тогда как во всех остальных местах кода сначала идет цветовой код, затем имя
2. **Несогласованность**: Сравните со строкой 123:
   ```cpp
   sprintf(buf, "%s%s отменил$G вас из темницы.%s", kColorBoldGrn, GET_NAME(ch), kColorNrm);
   ```
   Здесь порядок правильный: цвет → имя → сброс цвета

3. **Потенциальная уязвимость**: Если `GET_NAME(ch)` вернет строку, содержащую спецификаторы формата (например, `%s`, `%n`, `%x`), это может привести к format string уязвимости.

## Воспроизведение

Хотя напрямую эксплуатировать это сложно (так как имена игроков обычно валидируются), возможны следующие сценарии:

1. Если в имени каким-то образом окажутся символы `%`:
   ```
   Имя игрока: "Admin%s%s%s"
   Результат sprintf: "Admin%s%s%sНеопределенные значения из стека..."
   ```

2. Несогласованность приведет к тому, что цветовой код будет напечатан как часть текста:
   ```
   Ожидается: "\033[1;31mAdmin отправил вас в темницу.\033[0m"
   Получается: "Admin\033[1;31m отправил вас в темницу.\033[0m"
   ```

## Решение

Исправить порядок аргументов для согласованности с остальным кодом:

```cpp
// Было:
sprintf(buf, "%s%s отправил$G вас в темницу.%s", GET_NAME(ch), kColorBoldRed, kColorNrm);

// Должно быть:
sprintf(buf, "%s%s отправил$G вас в темницу.%s", kColorBoldRed, GET_NAME(ch), kColorNrm);
```
